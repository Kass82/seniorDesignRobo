Running Ubuntu in VM is very NOT recommended.
Developing in a simulated OS (VM) creates problems when running it on the RoboSub, which uses an actual live (R)OS.

Windows and Kubuntu dual boot with a single storage device:
*Please research if your device can contain multiple storage devices, its probably easier/better to just have another separate drive for a new OS. Or just get an external drive is fine too tbh.
Before starting, it is a good idea to have a backup of your drive in a separate location so if you mess up, youc an wipe the drive and clone it back.
    Prepare the ISO:
        Download the required Kubuntu (22.04.5 is the latest LTS to use with the Orin)
        FOR USB:
            Download a media creation tool (Rufus)
            Rufus should find the USB (or you choose the USB to use as Kubuntu boot)
            Locate the ISO for Rufus.
            Click start. There might be warnings or whatever, just continue or use default/recommended option.
            Finished, leave the USB connected.
        NO USB:
            You can follow the video https://www.youtube.com/watch?v=Ja3LCTsB7M0
            Like in ***, create a partition and allocate to FAT32 with enough space to contain the Kubuntu ISO. At least 5GB should be enough. Take note of the size.
            Right click ISO -> Open -> Copy all into the new partition.
            Run diskpart
            In diskpart run:
                list disk -> sel disk [###] (where [###] is the number of the particular disk that you created partitions, probably disk 0) -> list part -> sel part [###] (where [###] is the number of the partition you created for the boot, probably Partition 5. It should be a primary partition with the noted size)->help set id
            Scroll up and find GUID for EFI System Partition, copy the GUID
                set id=[GUID] (where [GUID] is the copied GUID
            Success, the boot partition should now be EFI System Partition in disk management.
    Ensure the storage device has sufficient space to keep your Windows install alongside the Kubuntu install.
        At least 50GB should be enough for the Kubuntu install. If running out of space, you're gonna have to physically open up your device and put some more storage in there.
    It's easier to start with a Windows, and then add the Kubuntu for dual booting.
    ***In Windows, open Disk Management. Identify the main storage partition (not EFI or recovery, it should be the larger partition).
        Right click -> split volume -> choose the size for the Kubuntu install.
        Leave this new partition unallocated.
    Restart the system. Spam the button to bring up BIOS (look it up for your device, probably F2 or F8 or something). Find the boot menu and move the device that contains the Kubuntu boot to be the first one loaded. Save changes. You should boot into Kubuntu (Try or Install Kubuntu).Proceed in Kubuntu installation.
    WHen selecting partition to put the Kubuntu install:
    DO NOT create over the whole drive
        In Kubuntu 22, you must select custom and initialize the unallocated space created earlier, with the mount point at /
        In newer versions, you can just straight up select the empty space and it will plcae Kubuntu in there.
    It should show what the drive looks like after installing Kubuntu. Make sure there are still parts that contains your Windows Install as well as the Kubuntu. Make sure Kubuntu is not a whole drive or takes up a suspiciously large amount of space.
    Kubuntu is installed, when (re)starting the system, GRUB will open where you can choose to launch Kubuntu or Windows.

Installing Gazebo Fortress
[instruction]

    Running Gazebo Fortress on Wayland
    https://gazebosim.org/docs/latest/troubleshooting/#wayland-issues
    Note that this link is to the latest version, but the fix does indeed apply to Fortress.
        Fix application launched via desktop/applications list
            Copy usr/share/applications/ignition-gazebo6.desktop to {home}~./local/share/applications and edit Exec line to add env QT_QPA_PLATFORM=xcb such that it becomes:
            ...
            Exec=env QT_QPA_PLATFORM=xcb /usr/bin/ign gazebo...
            ...
        Fix when launched via terminal
            Create an alias for ign gazebo
            In {home}~/.bashrc
            Add: alias 'ign-gazebo'='QT_QPA_PLATFORM=xcb ign gazebo'
            Now, whenever you would run ign gazebo, use ign-gazebo (e.g., ign-gazebo shapes.sdf)

Additional Wayland specific issue
    Running the Driver Manager via System Settings crashes
    https://bugs.launchpad.net/ubuntu/+source/kubuntu-driver-manager/+bug/1967469
    Go to /usr/bin/kubuntu-driver-manager and comment out 'export DISPLAY=:0'
    OR
    Instead of commenting out, run 'echo ${DISPLAY}' in the terminal, note the number it gives (e.g., :1), and change the number in the same file above (e.g., export DISPLAY=:1)
